## Design Context

### Users
Professionals and developers on macOS who write extensively — emails, documents, code, reports. Wren lives silently in the menu bar and activates only when needed. The user's context is "typing in any app and wanting to go faster." They value speed, accuracy, and zero distraction.

### Brand Personality
Warm, essential, confident, and discreet. Wren is a tool that feels like an extension of the user's own typing — not an AI assistant demanding attention. The tone is precise but not cold, friendly but not chatty. The UI should feel at home on macOS: native, lightweight, respectful of system conventions while having its own warm character.

### Aesthetic Direction
- **Theme**: Both light and dark modes, with warm-tinted neutrals (display P3).
- **Visual tone**: Clean, minimal, warm. Rounded but not pill-like. Flat surfaces with subtle depth only where functional.
- **Anti-reference**: Avoid AI-chatbot aesthetic (glassy modals, gradient text, purple/cyan accents). Avoid overly skeuomorphic textures. Avoid generic system defaults.
- **Key surface**: Menu bar popover as primary interface — must feel fast, lightweight, and native.

### Design Principles
1. **Native-first**: Respect macOS conventions. Use SwiftUI + AppKit where appropriate. Menu bar app should feel like it belongs.
2. **Discreet by default**: Wren finishes your sentences — the UI stays out of the way. No splash screens, no onboarding popups unless critical.
3. **Warm precision**: Warm-tinted surfaces (creamy light / warm dark) differentiate from sterile system UIs. Typography and spacing should feel generous but tight.
4. **One job, well**: Inline completion + typo fix. Don't bloat the feature set. Every settings tab must justify its existence.
5. **Performance is UX**: Ghost text overlay must appear within 50ms. The menu bar popover opens instantly. Every millisecond of perceived latency is a failure.
